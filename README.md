项目名称：CPP 高并发抢票引擎（研发中）
CPP High-Concurrency Ticketing Engine (Alpha Stage)

📖 项目简介
CPP 抢票程序是一个正在研发中的高性能自动化票务获取工具，旨在解决热门演出、赛事及交通票务在高峰期的并发访问难题。本项目采用 C++ 核心算法 + Python 调度层 的混合架构，致力于在保证极致速度的同时，提供安全、透明且可扩展的用户体验。

⚠️ 状态提示：本项目目前处于 Alpha 研发阶段。核心功能正在逐步完善，架构设计正在从“单体脚本”向“模块化安全架构”迁移。

🚀 核心特性（规划与研发中）
高性能并发内核
基于 C++ 重构核心请求模块，利用异步 I/O 模型实现毫秒级响应。
智能线程池管理，自动适配网络环境，最大化吞吐量。

智能设备指纹与授权
基于硬件特征的轻量级设备指纹生成算法，防止恶意滥用。
正在研发去中心化的授权验证方案，替代现有的中心化硬编码验证，提升系统鲁棒性。

用户友好交互
提供清晰的命令行交互界面（CLI），实时反馈网络状态与任务进度。
完善的错误处理机制，区分网络波动、库存不足与授权异常。

⚠️ 安全与免责声明

2026-3-27 更新
🔥 核心新增 / 升级功能（最重要）
1. 新增 网络精准时间同步
旧版：纯用本地电脑时间，容易不准、抢票慢
新版：对接阿里云官方时间接口，时间更精准，抢票成功率更高；失败自动 fallback 本地时间
2. 新增 自动答题功能
新版内置答题题库，登录后自动完成 CPP 答题验证
旧版完全没有，遇到答题会直接卡住 / 失败
3. 新增 多线程并发抢票
旧版：单线程慢慢抢，速度慢
新版：8 线程同时抢票，大幅提升成功率
加了线程安全锁，防止重复下单
4. 签名算法升级（更安全、更适配新版接口）
5. 移除本地 Token 存储 → 更安全
旧版注释写了 “强制每次重新登录”，新版彻底实现
不生成任何本地 token 文件，隐私更安全
🛠️ 体验与稳定性优化
6. 异常处理全面加强
新版几乎所有网络请求都加了容错
接口失败、格式错误、网络超时都不会直接崩溃
7. 界面输出更友好
加了分割线、进度条、倒计时
抢票状态实时刷新：抢票中... 尝试次数: xxx
成功后显示完整订单信息
8. 时间调整功能简化
旧版：手动输入毫秒调整
新版：固定 0 毫秒，避免误操作导致抢票失败
9. 新增 “按任意键退出”
结束后不会直接闪退，方便查看结果

🧱后续规划
1. 搭建云端数据库，在用户明确授权后，加密上传账号密码，构建统一抢票账号池。
2. 开发云端调度接口，实现抢票账号统一管理与状态标记，区分待抢、抢票中、已成功状态。
3. 客户端对接调度服务，单个账号抢购成功后，自动从数据库获取未购票账号继续抢票。
4. 联动全国多节点客户端，实现多地区、多账号、多线程并发抢票，大幅提升整体抢票成功率。
